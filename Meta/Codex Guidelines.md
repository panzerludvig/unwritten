# Codex Guidelines

Guidelines for maintaining the Codex so it stays useful as the project grows.

---

## Purpose

The Codex is a concise index of principles and standards, not a place for detail. It should be readable in one sitting and tell Claude — or any collaborator — how this project works at a high level.

---

## Structure

Each entry in the Codex should:
- State the principle or standard in a few sentences
- Link to a dedicated note where the detail lives
- Never grow long enough to require scrolling past

If an entry has no dedicated note yet, create a stub and link to it rather than expanding the Codex entry inline.

---

## Adding an Entry

1. Write the entry in the Codex — one short paragraph maximum
2. Create a dedicated note in the appropriate folder
3. Link the entry to that note
4. Commit both files together

---

## Reviewing the Codex

When a principle is questioned or a standard changes, update the dedicated note first, then revise the Codex entry if the summary no longer holds. Log the reasoning in [[Dev Log/Log]].
