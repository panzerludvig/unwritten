# Working with Obsidian

Guidelines for keeping this vault useful and consistent.

---

## Note Structure

- One topic per note. If a note is trying to cover two things, split it.
- Start every note with a single sentence describing what it is.
- Use headers to divide sections, not bold text.
- Link liberally with `[[double brackets]]` — prefer linking to duplicating information.
- Leave `_To be filled in._` as a placeholder rather than omitting a section.

---

## Folders

| Folder | Purpose |
|--------|---------|
| Design | Game design: mechanics, systems, loops, NPCs |
| Technical | Architecture, stack, tooling, LLM integration |
| Dev Log | Chronological progress notes |
| Assets | References to art, audio, and other files |
| Meta | Notes about the project process itself |

---

## Staying Current

- Log meaningful progress in [[Dev Log/Log]].
- When a decision is made, write it down before moving on.
